# 基于 Double DQN 的智能仓储机器人路径规划与货架拣选策略设计报告

- **直观测试（加载已有权重）**：如果你想直接利用已经存在的 `dqn_model.pth` 模型权重来直观展示智能体的寻路效果，请执行：

  该脚本会自动加载模型权重，并在终端界面动态渲染网格，以动画的形式一步步展示机器人（`[R]`）避开障碍物（`X`）寻找目标（`T`）的过程。最终也会在本地保存一张静态轨迹图 `test_evaluation_path.png`。

## 1. 问题描述与建模

### 1.1 仓库环境设定
本任务构建了一个 10×10 的二维网格仓储世界，起点固定在左上角 `(0, 0)`，目标货架设定在右下角 `(9, 9)`。同时，在网格中设置了若干障碍物（如 `(2, 2)`, `(5, 5)` 等），模拟真实的仓储隔离区与货架阻挡。

### 1.2 状态空间设计
机器人的状态被定义为其在网格中的归一化坐标表示 `(x / 9.0, y / 9.0)`，这是一个连续的 2 维向量，方便送入神经网络中进行处理并加速收敛。

### 1.3 动作空间设计
动作空间是一个离散的集合，包含 4 个基本动作：
- `0`: 上移 (Up)
- `1`: 下移 (Down)
- `2`: 左移 (Left)
- `3`: 右移 (Right)

### 1.4 奖励函数设计
为了促使机器人以最短路径到达目标并且避免碰撞，我们设计了如下奖励机制：
- 每移动一步：奖励 `-1.0`（鼓励寻找最短路径）。
- 撞到障碍物或边界：奖励 `-10.0`，且**不会结束回合**（为了便于在早期训练阶段有更多探索终点的机会）。
- 到达目标货架：奖励 `+100.0`，且结束该回合。

## 2. Double DQN 算法原理简述
传统的 DQN 会在计算目标 Q 值时直接使用目标网络的 `max` 操作，这通常会带来“过估计”问题（Overestimation）。
Double DQN (DDQN) 提出将动作的选择和动作的评估解耦：
1. **动作选择**：使用当前的主网络（Online Network）来选取下一个状态下的最优动作 $a^*$。
2. **动作评估**：使用目标网络（Target Network）来评估该动作 $a^*$ 的 Q 值。
公式为：
$Y_t = R_{t+1} + \gamma Q_{target}(S_{t+1}, \arg\max_a Q_{online}(S_{t+1}, a))$。
这一改进可以有效抑制过度乐观的 Q 值估计，使学习过程更加稳定。此外，我们也保留了经验回放（Experience Replay）和目标网络（Target Network）两大核心机制。

## 3. 实现细节
- **网络结构**：使用了三层全连接层 (MLP)。输入层为状态维度 `2`，隐藏层节点数为 `128`，输出层为动作维度 `4`。激活函数为 ReLU。
- **超参数设置**：
  - 学习率 (Learning Rate): `1e-3`
  - 折扣因子 ($\gamma$): `0.99`
  - $\epsilon$ 贪心探索：初始值 `1.0`，结束值 `0.01`，衰减率 `0.995`
  - 经验池大小 (Buffer Size): `10000`
  - 批量大小 (Batch Size): `64`
  - 目标网络同步频率: 每 `10` 个优化步同步一次。

## 4. 实验结果
### 4.1 训练曲线
在 `1000` 个回合的训练中，机器人的平均奖励和到达成功率在第 `150` 回合左右开始显著上升，在第 `300` 回合后成功率稳定在 `1.0`（100%），并且回合所用步数收敛到稳定值（约为 `18` 步左右）。
- 图表参考：`training_curves.png`

### 4.2 典型路径可视化
经过训练后，将 $\epsilon$ 设置为评估模式（纯贪心策略）。机器人从 `(0, 0)` 出发，能够智能地绕开前方的障碍物，形成了一条无碰撞且非常高效的最短路径到达 `(9, 9)`。
- 图表参考：`evaluation_path.png`

## 5. 分析与讨论
### 5.1 学习是否收敛？
是的，学习完全收敛。随着 $\epsilon$ 的降低，模型从完全随机探索顺利过渡到有效利用经验，奖励值和步数均达到了平稳最优状态。

### 5.2 改进方法对结果的影响
本次实验中实现了 **Double DQN** 这一改进扩展项。相比于基础 DQN 容易在迷宫问题中产生次优路线由于 Q 值过估计而导致来回震荡，Double DQN 通过解耦动作选择和评估，确保了目标值的稳定性。实验表明，经过短时间（约200次）回合即可突破障碍区找到稳定可行路径，收敛速度和策略稳定性都得到了极大的提升。通过环境逻辑改进（越界不直接截断），更是大大加速了探索过程。
